# Final Project Brief

The final project is where students turn the lesson sequence into something
useful. The goal is not to build a huge application. The goal is to practice
choosing data types, organizing code, handling errors, and explaining design
decisions.

## Project Goal

Build a small Python program that solves a real or realistic problem.

Good project ideas include:

- a study tracker
- a habit tracker
- a flashcard quiz tool
- a reading list manager
- a simple contact manager
- a budget or expense tracker
- a local chatbot tutor
- a document search demo
- a lightweight AI helper for a repeated task

## Required Features

Your project should include:

- at least one list, tuple, set, or dictionary chosen for a clear reason
- at least two functions
- at least one class or a short written explanation of why a class is not needed
- file reading or file writing
- error handling with `try` and `except`
- comments that explain important design choices
- a short `README` or docstring explaining how to run the project

## Optional Advanced Features

Choose one or more if they fit your idea:

- read or write JSON
- use environment variables for configuration
- call a public API
- call Ollama locally
- call the OpenAI API with a guarded example
- create a simple chatbot loop
- create a tiny embedding or keyword search demo
- build a toy RAG pipeline
- add a simple agent-style tool loop

## Design Questions

Before writing code, answer these questions in comments or notes:

- What problem does this project solve?
- What data does the program need to store?
- Which data types did you choose, and why?
- What can go wrong while the program runs?
- Which part of the code would you improve if you had more time?

## Suggested Project Structure

For a small project, one file is fine:

```text
my_final_project.py
```

For a larger project, use a folder:

```text
my_final_project/
    README.md
    main.py
    data.json
```

## Review Checklist

Before sharing the project:

- Run the program from start to finish.
- Try one invalid input or missing-file case.
- Check that no API keys or secrets are saved in the code.
- Make sure generated files are either ignored or safe to share.
- Read the code once as if someone else wrote it.

## Reflection Prompt

At the end, write a short reflection:

- What did this project teach you?
- What part was hardest?
- What design choice are you most confident about?
- What would you build next?
